# Guía de capturas para la evidencia del examen

Abra `docs/evidencia_examen.docx` en Word o súbalo a Google Docs. Luego reemplace cada recuadro por las capturas reales.

## Capturas mínimas recomendadas

1. **Creación de base de datos en la nube**: panel del proveedor cloud con la base creada.
2. **Script SQL ejecutado**: consola SQL mostrando que `migrations.sql` se ejecutó.
3. **Tabla songs creada**: estructura de la tabla con los campos.
4. **Datos de prueba**: registros insertados por `seed.sql`.
5. **Variables de entorno**: configuración de `DB_HOST`, `DB_DATABASE`, `DB_USERNAME` sin mostrar la contraseña completa.
6. **Aplicación en la nube**: URL pública funcionando.
7. **Formulario de inserción**: pantalla `Nueva canción`.
8. **Cálculo automático**: vista del cálculo JavaScript de ganancia y popularidad.
9. **Listado/búsqueda**: búsqueda por género, artista o canción.
10. **Código MVC/ORM**: carpetas `Controllers`, `Models`, `Views` y archivo `app/Core/ORM/Model.php`.

## Recomendación

No muestre contraseñas reales en las capturas. Si aparece una contraseña, difumine o tape esa parte antes de entregar.
